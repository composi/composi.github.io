Init

Preparar para el Inicio

Cuando pasas un programa a la función run, el entorno de ejecución toma el valor devuelto por el método init para usarlo para configuracón. Si su programa tiene estado, entonces el método init lo devolverá envuelto en una matriz. Si deseas ejecutar un efecto al inicio, también lo incluiría como el segundo valor de la matriz devuelto por init. Un efecto es solo una función. Un efecto podría ser un temporizador de algún tipo, o una solicitud de datos. Un efecto es siempre asincrónico, ejecutándose independientemente del resto del programa. La mayoría de las veces, su inicio solo volverá al estado. Aun así, siempre debes encerrarlo entre llaves.

const program = {
  init() {
    return [estado]
  },
  view() {},
  update() {}
}

Cuando quieres operar un efecto durante el inicio del programa, siempre va a ser el segundo valor en el vector que devuelve init:

const program = {
  init() {
    return [estado, efecto]
  },
  view() {},
  update() {}
}

Ejecutar Efecto al Iniciarse

Por proporcionar un efecto como el segundo valor en la matriz que devuelta por init, puedes ejecutar en efector al iniciar el program. Si prefieres, puedes en vez usar el método subscriptions del programa para lo mismo. Lee la documentación de subscriptions para detalles de cómo usar los.

Aquí es un escrito que muestra cómo usar un efecto en el método init, mientras también devuele estado:

// Contado para la vista:
function Contador({estado, send}) {
  return (
    <p>
      {
        estado
      }
    </p>
  )
}

// Un efecto que ejecutar al iniciar:
function effect() {
  setTimeout(() => {
    alert('¡El efecto funciona!')
  }, 3000)
}

// Asembla el programa:
const programa = {
  init() {
    // Devolver el efector con el estado:
    return ['un valor para el estado', effect]
  },
  view(estado) {
  render(<Contador {...{estado}} />, 'section')
  },
  update() {}
}

// Run program:
run(programa)
