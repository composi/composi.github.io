Init

Preparing for Startup

When you pass a program to the run function, the runtime takes the value returned by the init method to use for setting things up. If your program has state, then the init method must return it wrapped in an array. If you want to run an effect at startup, you would also include that as the second value of the array returned by init. An effect is just a function. An effect could be a timer of some kind, or a request for data. An effect is always asynchronous, running independently of the rest of the program. Most of the time your init will only return state. Even so, you must always enclose it in braces.

const program = {
  init() {
    return [state]
  },
  view() {},
  update() {}
}

When you do want to run an effect at startup, it will always be the second value in the array that init returns:

const program = {
  init() {
    return [state, effect]
  },
  view() {},
  update() {}
}

Run Effect at Startup

And here's a snippet that shows how to use an effect in the init method, returning it with the state:

// Counter for view:
function Counter({state, send}) {
  return (
    <p>
      {
        state
      }
    </p>
  )
}

// An effect to run at startup:
function effect() {
  setTimeout(() => {
    alert('The effect ran!')
  }, 3000)
}

// Assemble program together:
const program = {
  init() {
    // Return the effect with the state:
    return ['Default State Value', effect]
  },
  view(state) {
  render(<Counter {...{state}} />, 'section')
  },
  update() {}
}

// Run program:
run(program)
