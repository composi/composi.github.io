State

State for Functional Components

Functional components are stateless. You pass data to them as a prop and that gets used during the render. This means that if you pass the same data to a functional component, it will render the exact same way each time. This makes rendering predictable and easier to troubleshoot.

In contrast, class-based components can encapsulate local state and behaviors that make management and reuse easy.

@composi/core provides only functional components with a virtual DOM. Composi does not provide a way to do state management for functional components. It's @composi/datastore. You'll need to install.

npm i -D '@composi/core'

After that, you need to import it into your project.

import { h, render } from '@composi/core'
import { DataStore } from '@composi/datastore'

After you have DataStore imported, you can create a dataStore. You do this by providing it some data. This will be an object literal with one property: state. You data will be the value of the dataStore's state. The state can be a simple value or an array or object. You will then use the dataStore all the ultimate source of truth for the functional component's state. You do this by passing the dataStore state to the function as a property. DataStores are reactive. They have an internal observer that watches for changes and then fires an event. You can define a watcher on the dataStore with an appropriately named event and have it re-render the functional component. Then, when you need to update state, you use the setState method on the dataStore to do so. This causes the watcher to react to the state changes and re-render the functional component. Job done.

Here is an example of creating a dataStore to use with a component:

import { h, render } from '@composi/core'
import { DataStore } from '@composi/datastore'

// Create a dataStore:
const dataStore = new DataStore(['Apples', 'Oranges', 'Bananas'])

// Define refs object to hold reference to input:
const refs = {}

// Define app for holding events, etc.
const app = {
  init(base) {
    refs.input = base.querySelector('input')
    refs.input.focus()
  },
  addItem() {
    const value = refs.input.value
    if (value) {
      // Add value to dataStore:
      dataStore.setState(prevState => {
        prevState.push(value)
        return prevState
      })
      // Clear the input's value,
      // and give it focus:
      refs.input.value = ''
      refs.input.focus()
    }
  }
}

// Create a list component and pass the dataStore to it as data:
function List({data}) {
  return (
    <div onmount={base => app.init(base)}>
      <p>
        <input type="text" />
        <button onclick={() => app.addItem()</button>}</button>}>Add</button>
      </p>
    </div>
  )
}

// Render the list and pass in the dataStore state as a prop:
let list = render(<List data={dataStore.state} />, 'section')

// Set up a watcher and use it to update the component:
dataStore.watch('update-list', () => {
  list = render(<List data={dataStore.state} />, 'section', list)
})

To learn more about how to use DataStore, read the documentation.
