State

State Management for Functional Components

Class components enable local state. However, @composi/core is a functional libray. By their very nature, functional components are stateless. You will certainly find yourself needing local state for your components. @composi/core provides a means of state management for functional components through its runtime. This is a simplified implementation of TEA: The Elm Architecture.

SVU

Every runtime program breaks its functionality down to three parts:

  • State: any valid JavaScript type
  • View: the visual rendering of the program's state
  • Update: actions that return a new version of state

Because state management is a complex subject and the runtime provides many features, we have a dedicated section for it: Runtime
